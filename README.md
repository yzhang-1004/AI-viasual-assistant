# 心灵树洞 — AI 视觉对话助手

> 一款以 1v1 视频通话为核心交互范式的 AI 情感陪伴应用，让 AI 不仅能"听"你说话，还能"看"到你的状态。

## 产品理念

「心灵树洞」聚焦**情感陪伴与轻度心理疏导**，摒弃传统"对话框+键盘"的聊天机器人形态，让用户感觉像是在和一位温暖的知心朋友打视频电话。

- **视频通话即视感**：全屏沉浸式界面，去聊天框化，用声音和画面交流
- **安全无审视（树洞效应）**：用户说的话**不在屏幕上显示文字**，消除"被记录/被评判"的心理负担
- **视听融合感知**：AI 不仅"听"内容，还能"看"画面（深夜昏暗灯光、疲惫神态等），给出更具同理心的回应
- **极速共情响应**：优化端云链路，实现打断式对话和毫秒级首字响应

## 界面与交互设计

### 视觉风格

- **色调**：暮光橘、暖琥珀、深栗色为主，森林绿点缀，营造温暖、私密的"树洞"氛围
- **背景**：缓慢漂浮的萤火虫粒子与暗部虚化光斑，增加空间感与治愈感
- **AI 形象**：不使用拟人化假人（避免恐怖谷效应），采用**"呼吸光晕"**作为 AI 的视觉化身——倾听时缓慢起伏，说话时加速脉动并产生涟漪

### 核心布局

```
┌─────────────────────────────────────────┐
│  [模式切换栏]  陪伴倾听 | 解惑 | 冥想 | 每日  │  ← 顶部
│                                         │
│  ┌──────────┐                           │
│  │ 摄像头小窗 │                           │  ← 左上角
│  └──────────┘                           │
│                                         │
│          ◎ 呼吸光晕 ◎                     │  ← 中央视觉焦点
│          （倾听/思考/说话态）               │
│                                         │
│        AI 回复文字（淡入淡出）               │  ← 中央下方
│                                         │
│     📷    [ 🎙 麦克风 ]    ✏️    📞      │  ← 底部操控区
└─────────────────────────────────────────┘
```

### 关键交互

- **用户的话不显示**：仅通过麦克风按钮的声波纹动画提供反馈，不在界面留下文字记录
- **AI 回复文字**：仅在 AI 说话时以打字机效果显示，说完后逐渐隐去，保持画面干净
- **视觉反馈映射**：AI 识别到用户环境较暗时，界面亮度自动微调变暗，体现"AI 在关注你"

## 四大功能模式

| 模式 | 定位 | AI 行为 |
|------|------|---------|
| **陪伴倾听**（默认） | 纯情绪价值，共情优先 | 极少建议，多用"嗯""我在听""那一定很难受"引导宣泄 |
| **解惑指路** | 理性分析，提供视角 | 引入 CBT 框架，帮助拆解问题，提供可执行建议 |
| **冥想放松** | 正念引导，缓解焦虑 | 语速变慢，引导 4-7-8 呼吸法，界面出现动态呼吸圈 |
| **每日力量** | 正向激励，建立韧性 | 基于近期情绪生成专属肯定语和微行动建议 |

---

## 核心特性

- **CosyVoice 流式 TTS**：接入灵积 CosyVoice WebSocket，边生成边播放，自然音色秒级响应
- **AI 打断机制**：AI 说话时点击按钮即可打断，模拟真人对话的自然节奏
- **端侧视觉感知**：face-api.js 端侧分析摄像头画面（光照、表情、是否出镜），隐私零泄露
- **语音回路防护**：TTS 朗读期间自动暂停语音识别，防止 AI 把自己的话当用户输入

## 快速开始

### 环境要求

- Node.js >= 18
- Chrome 浏览器（语音识别 API 支持）

### 安装与运行

```bash
# 进入项目目录
cd mind-tree-hole

# 安装依赖
npm install

# 配置 API Key（二选一）
# 方式一：复制 .env.example 为 .env 并填入 Key
cp .env.example .env
# 方式二：启动后在界面设置面板中填入

# 启动开发服务器
npm run dev
```

### 获取 API Key

1. 访问 [阿里云百炼控制台](https://bailian.console.aliyun.com/#/api-key)
2. 注册/登录并创建 API Key
3. 将 Key 填入 `.env` 文件或应用内设置面板

## 架构设计

### 端云协同与成本控制

多模态大模型调用成本极高，本项目采用**"端侧预处理 + 按需触发云端"**的降维策略：

| 层级 | 策略 | 效果 |
|------|------|------|
| **视觉降维** | 不传视频流，端侧每 2s 截帧分析亮度+表情，转为文本 `[视觉上下文：环境昏暗，用户面容疲惫]` 注入 Prompt | 成本降至多模态的 1/10 |
| **多级路由** | L0 规则引擎（0成本 <100ms）→ L2 云端文本 LLM（标准成本 1-2s）→ L3 多模态（仅用户主动触发） | 按需调用，避免浪费 |
| **语音过滤** | 端侧 VAD 静音期不调用 ASR；语气词（嗯/啊/哦）直接丢弃不触发 LLM | 降本约 30% |

### 极速响应链路

```
用户停顿 0.8s → 预测性预填（不等说完即发送）
     → LLM 流式输出首个分句
     → 立即喂入 CosyVoice TTS
     → 首段音频秒级播放
     → 光晕切换为"说话态"涟漪
```

### 端侧视觉感知

```
摄像头 → 每2s截帧到隐藏canvas → 亮度计算 + face-api.js人脸/表情检测
     → "[视觉上下文：环境光线偏暗，用户面带笑容]"
     → 注入 System Prompt → 纯文本 LLM（低成本）
```

全程不传图到云端，隐私零泄露。
