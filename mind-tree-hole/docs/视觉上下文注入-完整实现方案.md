# 视觉上下文注入 — 完整实现方案

## 目标

让 AI 在不调用云端多模态大模型的前提下，通过端侧轻量分析感知用户状态（光照、是否出镜、大致情绪），生成文本描述注入 System Prompt，提升 AI 回应的共情质量。

## 核心设计原则

- **纯端侧**：不传图到云端，不增加 API 成本
- **渐进式**：分两期，一期能跑通，二期加智力
- **新依赖零门槛**：一期零依赖，二期 face-api.js 仅 ~200KB（gzip ~60KB）

---

## 整体架构

```
Camera Video Element
    │
    ├──→ 屏幕显示（已有：UserCameraWindow）
    │
    └──→ useVisionContext（新建 Hook）
            │
            ├── 每 2s 截帧到隐藏 canvas
            │
            ├── 一期：亮度分析 → 环境光线描述
            │
            ├── 二期：人脸检测 → 是否出镜 + 表情趋势
            │
            └──→ setVisionContext() → store
                                       │
                                       ▼
                              buildSystemPrompt() → LLM
```

## 一期：环境光照感知（极简 MVP）

只做亮度检测，跑通整条链路。

### 数据流

```
video → canvas(80×60) → getImageData → 加权亮度均值 → 映射文本
```

### 亮度计算

| 参数 | 值 |
|------|-----|
| canvas 尺寸 | 80×60（4800 像素，足够判断） |
| 采样策略 | 每 4 像素取 1 个（约 1200 次运算） |
| 加权公式 | `L = 0.299×R + 0.587×G + 0.114×B`（ITU-R BT.601） |
| 更新频率 | 每 2 秒一次 |
| 变化阈值 | 变化 < 10% 时跳过，避免频繁更新 |

### 亮度 → 描述映射

| 亮度范围 | 描述文本 |
|----------|----------|
| 0–50 | `[视觉上下文：环境光线很暗]` |
| 50–120 | `[视觉上下文：环境光线偏暗]` |
| 120–255 | `[视觉上下文：环境光线充足]` |

### 改动清单

| 文件 | 操作 | 行数 |
|------|------|------|
| `src/hooks/useVisionContext.ts` | **新建** | ~55 行 |
| `src/components/video/UserCameraWindow.tsx` | 改为 `forwardRef`，暴露 video | ~5 行改动 |
| `src/components/layout/FullScreenCall.tsx` | 获取 videoRef，调用 hook | ~5 行新增 |

### 零改动项
- `stores/appStore.ts` — `visionContext` + `setVisionContext` 已存在
- `hooks/useConversation.ts` — 第 85 行已读取 `visionContext` 拼入 prompt
- `utils/prompts.ts` — `buildSystemPrompt()` 第 10 行已拼接 `visionContext`

### 效果示例

```
// 用户在昏暗环境说"睡不着"
System Prompt:
  你是一位温暖的朋友...
  [视觉上下文：环境光线很暗]
  "睡不着"

AI 看到"光线很暗" → 可能回应：
"夜深了还没睡...是心里有事吗？我在这听着"
```

## 二期：人脸感知（真正的价值点）

在一期基础上叠加 face-api.js，实现"AI 能看到你笑了 / 你没在看屏幕 / 你出画了"。

### 新增依赖

```
face-api.js          (~200KB gzip: ~60KB)
@tensorflow/tfjs     (~800KB gzip: ~200KB，face-api.js 的运行时依赖)
```

**加载策略**：动态 import，仅在摄像头开启时按需加载，不阻塞首屏。

### 检测维度

| 维度 | 检测内容 | 生成文本示例 |
|------|----------|-------------|
| 是否出镜 | 人脸存在/消失 | `[视觉上下文：用户离开了画面]` |
| 情绪趋势 | 开心/中性/难过 | `[视觉上下文：用户看起来情绪低落]` |
| 头部姿态 | 正对/侧对/低头 | `[视觉上下文：用户似乎低下了头]` |
| 环境光照 | 复用一期逻辑 | `[视觉上下文：环境昏暗]` |

### 复合描述范例

```
// 多个检测结果合并为一条：
[视觉上下文：环境光线偏暗，用户面容疲惫，情绪低落]
```

### 二期改动增量

| 文件 | 操作 |
|------|------|
| `useVisionContext.ts` | 新增 face-api 初始化 + 表情/人脸检测逻辑 |
| `public/models/` | 放置 face-api 的 tiny 模型文件（4 个小文件） |
| 其余文件 | 一期已就绪，无额外改动 |

### 为什么二期才有实质意义

- 光照信息太单一，AI 多知道"亮/暗"对回复提升不大
- **情绪趋势**才是真正的价值——"你看起来很低落" vs "你看起来很开心"，AI 回应的方向完全不同
- **是否出镜**也很关键——用户起身离开的画面信息，文字对话完全无法传达

## 具体代码设计

### `useVisionContext` Hook 接口

```ts
interface UseVisionContextOptions {
  videoRef: React.RefObject<HTMLVideoElement | null>
  enabled: boolean        // cameraEnabled 时启用
  intervalMs?: number     // 截帧间隔，默认 2000
  enableFace?: boolean    // 二期开关，默认 false
}

function useVisionContext(options: UseVisionContextOptions): void
```

### Hook 内部流程

```
useEffect(() => {
  if (!enabled) return

  // 创建隐藏 canvas（不插入 DOM）
  const canvas = document.createElement('canvas')
  canvas.width = 80; canvas.height = 60
  const ctx = canvas.getContext('2d')!

  // 二期：按需加载 face-api
  if (enableFace) {
    loadFaceApi()          // 动态 import + 模型加载
  }

  const timer = setInterval(() => {
    // 1. video 就绪检查
    // 2. drawImage 截帧
    // 3. 计算亮度
    // 4. 二期：执行人脸检测
    // 5. 组装描述文本
    // 6. 与上次对比，变化 > 阈值才 setVisionContext
  }, intervalMs)

  return () => {
    clearInterval(timer)
    setVisionContext('')   // 退出摄像头时清理
  }
}, [enabled])
```

## 风险与限制

| 风险 | 说明 | 对策 |
|------|------|------|
| face-api.js 体积 | tfjs ~800KB | 动态 import，只首加载一次 |
| 端侧性能 | 人脸检测耗时 | 频率降到 3s 一次，tiny 模型极轻 |
| 表情不准 | 端侧模型精度有限 | 只分"正面/负面"二类，不追求细粒度 |
| 隐私 | 截帧数据 | canvas 不插入 DOM，用完即弃，不上传 |
