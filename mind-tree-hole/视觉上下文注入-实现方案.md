# 视觉上下文注入 — 实现方案

## 目标

实现"端侧截帧 → 分析亮度 → 生成文字 → 注入 Prompt"的完整链路，让 AI 在不调用多模态大模型的前提下，感知用户的环境光照状态。

---

## 整体数据流

```
摄像头 video 元素
    │
    ├──→ 屏幕显示（UserCameraWindow 已有）
    │
    └──→ 每 2s 截一帧到隐藏 canvas
              │
              ▼
         getImageData() 取像素
              │
              ▼
         计算平均亮度（加权公式）
              │
              ▼
         映射为中文描述
         "环境明亮" / "环境昏暗" / "环境较暗"
              │
              ▼
         setVisionContext("[视觉上下文：环境昏暗]")
              │
              ▼
         buildSystemPrompt() 拼入 System Prompt
              │
              ▼
         发送给纯文本 LLM → AI 据此调整语气
```

## 详细设计

### 1. 新建 `src/hooks/useVisionContext.ts`

核心逻辑如下：

```ts
// 伪代码（不是最终实现，仅说明思路）

export function useVisionContext(options: {
  videoRef: React.RefObject<HTMLVideoElement | null>
  enabled: boolean          // 摄像头开启时才工作
  intervalMs?: number       // 截帧间隔，默认 2000
}) {
  useEffect(() => {
    if (!enabled) return

    const canvas = document.createElement('canvas')
    const ctx = canvas.getContext('2d')!
    // canvas 尺寸不用太大，80×60 足够计算亮度
    canvas.width = 80
    canvas.height = 60

    const timer = setInterval(() => {
      const video = options.videoRef.current
      if (!video || video.readyState < 2) return  // 视频未就绪

      // 1. 把当前帧画到 canvas
      ctx.drawImage(video, 0, 0, 80, 60)

      // 2. 取像素数据
      const imageData = ctx.getImageData(0, 0, 80, 60)

      // 3. 计算加权平均亮度
      const avgLuminance = calcAvgLuminance(imageData.data)

      // 4. 生成中文描述
      const desc = luminanceToDesc(avgLuminance)

      // 5. 写入 store
      useAppStore.getState().setVisionContext(desc)

    }, intervalMs)

    return () => {
      clearInterval(timer)
      useAppStore.getState().setVisionContext('')  // 清理
    }
  }, [enabled])
}
```

#### 亮度计算算法

```
加权亮度 = 0.299 × R + 0.587 × G + 0.114 × B  （ITU-R BT.601 标准）

遍历每个像素 → 累加 → 除以像素总数 → 得到平均值 (0-255)
```

#### 亮度 → 中文映射

| 平均亮度 | 分类 | 生成文本 |
|----------|------|----------|
| 0 – 50 | 很暗 | `[视觉上下文：环境光线很暗，用户可能在黑暗中使用]` |
| 50 – 130 | 偏暗 | `[视觉上下文：环境光线偏暗，整体氛围安静私密]` |
| 130 – 255 | 明亮 | `[视觉上下文：环境光线充足明亮]` |

#### 性能优化

- **缩小 canvas**：80×60 仅 4800 像素，计算量极小
- **跳帧采样**：每 N 个像素取 1 个（如隔 4 取 1），进一步降低到 ~1200 次运算
- **变化判断**：亮度变化 < 5% 时不更新 store，减少不必要的 re-render

## 生效路径验证

以"陪伴倾听"模式为例，假设用户在昏暗环境中说"我今天好累"：

```
1. useVisionContext 检测到亮度 45 → setVisionContext("[视觉上下文：环境光线很暗...]")
2. 用户说"我今天好累" → processUserInput()
3. buildSystemPrompt('companion', visionContext) 输出：

   "你是一位温暖、善解人意的朋友...
    [视觉上下文：环境光线很暗，用户可能在黑暗中使用]
    用户说：我今天好累"

4. LLM 看到"环境光线很暗" → 可能回应：
   "这么晚了还没休息吗？听起来你今天很累，我在这里陪着你..."
```

AI 的回应会自然地带上对环境的感知，不需要多模态模型。
