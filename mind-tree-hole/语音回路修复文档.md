# 语音回路修复文档

## 问题描述

用户说话后，AI 通过 TTS（浏览器 SpeechSynthesis）朗读回复，但浏览器语音识别（SpeechRecognition）仍在监听麦克风，导致 AI 的朗读内容被音箱播放后又被麦克风录入，识别成文字后当成用户输入再次处理，形成死循环。

**现象**：AI 回复后，会把自己的话也当成用户发言，反复触发对话。

## 根因分析

| 环节 | 说明 |
|------|------|
| 语音识别 | `SpeechRecognition` 设置了 `continuous: true`，持续监听麦克风 |
| TTS 朗读 | `speakText()` 调用 `SpeechSynthesis` 通过音箱播放 AI 回复 |
| 回路形成 | 音箱声音 → 麦克风录入 → 语音识别 → `processUserInput()` → 再次触发 LLM + TTS |

关键代码位置：`src/hooks/useConversation.ts`

```
音箱 ──音频──→ 麦克风 ──转录──→ SpeechRecognition.onresult
                                          │
                                    processUserInput()
                                          │
                                    streamLLM() + speakText()
                                          │
                                          └──── 循环 ────┘
```

## 修复方案

在 TTS 朗读期间**暂停语音识别**，朗读结束后再恢复。

### 修改文件

`src/hooks/useConversation.ts`

### 具体改动

#### 1. 新增状态标记（第 26 行）

```ts
const isTtsSpeakingRef = useRef(false) // TTS 朗读期间暂停语音识别，防止语音回路
```

#### 2. 新增两个辅助函数（第 28-49 行）

```ts
const pauseRecognitionForTTS = () => {
  isTtsSpeakingRef.current = true
  if (recognitionRef.current) {
    try { recognitionRef.current.stop() } catch { /* 忽略 */ }
  }
}

const resumeRecognitionAfterTTS = () => {
  isTtsSpeakingRef.current = false
  if (recognitionRef.current && useAppStore.getState().callStatus !== 'idle') {
    setTimeout(() => {
      try { recognitionRef.current?.start() } catch { /* 忽略 */ }
    }, 300)
  }
}
```

#### 3. L0 规则响应路径 — 朗读前后加保护

```ts
// 修改前
await speakText(l0Response)

// 修改后
pauseRecognitionForTTS()
await speakText(l0Response)
resumeRecognitionAfterTTS()
```

#### 4. LLM 流式响应路径 — 朗读前后加保护

```ts
// 修改前
await speakText(fullText)

// 修改后
pauseRecognitionForTTS()
await speakText(fullText)
resumeRecognitionAfterTTS()
```

#### 5. `onresult` 回调 — TTS 期间忽略识别结果

```ts
recognition.onresult = (event) => {
  // 新增：TTS 朗读期间忽略所有识别结果
  if (isTtsSpeakingRef.current) return
  // ... 原有逻辑
}
```

#### 6. `onend` 回调 — TTS 期间不自动重启

```ts
recognition.onend = () => {
  // 新增：TTS 朗读期间不自动重启
  if (isTtsSpeakingRef.current) return
  // ... 原有逻辑
}
```

## 修复效果

| 修复前 | 修复后 |
|--------|--------|
| AI 朗读时音箱声音被麦克风录入 | AI 朗读时语音识别暂停 |
| AI 把回复当成用户输入再次处理 | 语音识别只在用户真正说话时触发 |
| 形成死循环 | 正常对话流程 |

## 验证方式

1. 启动项目：`npm run dev`
2. 进入通话模式，对 AI 说话
3. 等待 AI 语音回复
4. 确认 AI 回复后不会再次触发对话
