# AI 视觉对话助手 - 项目功能结构文档

## 📋 项目概述

一个基于多模态大模型的 AI 视觉对话助手 Web 应用，支持摄像头视觉识别、语音对话、文字交互等功能。

**技术栈**：Next.js 16 + TypeScript + Tailwind CSS 4 + shadcn/ui + 阿里云灵积 DashScope

---

## 🏗️ 整体架构

```
前端 UI (React/Next.js)
    ↓
API Routes (Next.js Server Routes)
    ↓
DashScope API (阿里云灵积)
    ├── LLM: qwen-max (文字对话)
    ├── VLM: qwen-vl-max (视觉对话)
    ├── ASR: paraformer-v2 (语音识别)
    └── TTS: cosyvoice-v2 (语音合成)
```

## 🔧 核心功能模块

### 1. **灵积 API 客户端** (`src/lib/dashscope.ts`)

**职责**：封装所有阿里云灵积 API 调用

#### 功能列表：
- ✅ **LLM 文字对话** (`chatCompletion`)
  - 模型：qwen-max（可切换 qwen-plus/qwen-turbo）
  - 接口：OpenAI 兼容格式
  - 参数：messages, temperature, max_tokens, stream

- ✅ **VLM 视觉对话** (`visionChatCompletion`)
  - 模型：qwen-vl-max（可切换 qwen-vl-plus）
  - 支持：文本 + 图片多模态输入
  - 接口：OpenAI 兼容格式

- ✅ **ASR 语音识别** (`speechRecognition`)
  - 模型：paraformer-v2
  - 输入：Base64 编码的 WAV 音频
  - 输出：识别文本

- ✅ **TTS 语音合成** (`speechSynthesis`)
  - 模型：cosyvoice-v2
  - 输入：文本
  - 输出：WAV 音频 Buffer
  - 支持 12 种音色（longxiaochun、longlaotie 等）

- ✅ **工具函数** (`cleanResponse`)
  - 清理 AI 回复中的思维链标签 `<think>...</think>`

---

### 2. **视觉对话 API** (`src/app/api/vision-chat/route.ts`)

**职责**：处理视觉+文字对话请求，智能选择 LLM 或 VLM 模型

#### 工作流程：
```
请求 → 判断是否有图像 
       ├─ 有图像 → VLM (qwen-vl-max) → 视觉理解回复
       └─ 无图像 → LLM (qwen-max) → 纯文字回复
```

#### 功能特性：
- ✅ 自动选择模型（视觉/文字）
- ✅ 对话历史管理（保留最近 10 条）
- ✅ System Prompt 预设（智能助手人设）
- ✅ Base64 图片处理（自动添加 data URI 前缀）
- ✅ 思维链内容过滤
- ✅ 错误处理与状态码

#### API 接口：
```typescript
POST /api/vision-chat
Body: {
  message: string,          // 用户消息
  imageBase64?: string,     // Base64 图片（可选）
  history?: Array<{         // 对话历史（可选）
    role: 'user' | 'assistant',
    content: string
  }>,
  hasImage: boolean         // 是否包含图像
}

Response: {
  success: boolean,
  response: string,         // AI 回复
  mode: 'vision' | 'text'   // 使用的模式
}
```

---

### 3. **语音识别 API** (`src/app/api/asr/route.ts`)

**职责**：将音频转换为文字

#### 功能特性：
- ✅ 接收 Base64 音频数据
- ✅ 自动去除 data URL 前缀
- ✅ 调用 paraformer-v2 模型
- ✅ 返回识别文本

#### API 接口：
```typescript
POST /api/asr
Body: {
  audioBase64: string       // Base64 编码的 WAV 音频
}

Response: {
  success: boolean,
  text: string              // 识别结果
}
```

---

### 4. **语音合成 API** (`src/app/api/tts/route.ts`)

**职责**：将文字转换为语音

#### 功能特性：
- ✅ 文本长度限制（1024 字符）
- ✅ 调用 cosyvoice-v2 模型
- ✅ 默认音色：longxiaochun（温暖女声）
- ✅ 返回 WAV 音频流
- ✅ 支持语速调节

#### API 接口：
```typescript
POST /api/tts
Body: {
  text: string,             // 待合成文本
  voice?: string,           // 音色（可选）
  speed?: number            // 语速（可选，默认 1.0）
}

Response: Audio/WAV 二进制流
```

## 🎯 成本控制策略

### 已实现的优化：

1. **端侧 VAD**：仅在用户说话时调用 ASR，静默段不消耗 API
2. **智能帧采样**：可调节采样间隔，非对话时降低频率
3. **图像压缩**：发送前缩放至 512px + JPEG 质量 0.6
4. **场景变化检测**：画面无变化时不更新图像
5. **分级模型调用**：纯文字用 qwen-max（便宜），视觉用 qwen-vl-max
6. **上下文窗口管理**：对话历史只保留最近 10 条
7. **TTS 文本截断**：限制 1024 字符
8. **成本统计仪表盘**：实时显示调用次数和节省率

### 模型价格参考：

| 用途 | 模型 | 价格（元/千token） | 更便宜备选 |
|------|------|-------------------|-----------|
| LLM 对话 | qwen-max | 0.02 | qwen-plus (0.004) / qwen-turbo (0.001) |
| VLM 视觉 | qwen-vl-max | 0.02 | qwen-vl-plus (0.008) |
| ASR 识别 | paraformer-v2 | 0.02元/分钟 | paraformer-8k-v1 |
| TTS 合成 | cosyvoice-v2 | 0.01元/千字符 | sambert-zhichu-v1 |

## 🔄 数据流

### 文字对话流程：
```
用户输入文字 
  → 前端发送 POST /api/vision-chat
  → 后端调用 chatCompletion (qwen-max)
  → 返回 AI 回复
  → 前端显示 + TTS 语音播报
```

### 语音对话流程：
```
用户说话 
  → VAD 检测语音活动
  → 录音并转换为 WAV
  → 前端发送 POST /api/asr
  → 后端调用 speechRecognition (paraformer-v2)
  → 返回识别文本
  → 自动触发文字对话流程
```

### 视觉对话流程：
```
摄像头开启
  → 帧采样 + 场景变化检测
  → 图像压缩为 512px JPEG
  → 用户发送消息时附带图像
  → 前端发送 POST /api/vision-chat
  → 后端调用 visionChatCompletion (qwen-vl-max)
  → 返回 AI 视觉理解回复
  → 前端显示 + TTS 语音播报
```
